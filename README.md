Серверная часть интеллектуального помощника **ScanDiet**.  
Сервис получает данные о продукте по штрих-коду из базы [Open Food Facts](https://world.openfoodfacts.org/) (или другого провайдера), извлекает текст состава и с помощью нейросетевой модели распознаёт в нём **пищевые аллергены и компоненты, соответствующие диетическим ограничениям**.  
Модель – **BiLSTM + CRF**, обученная на русскоязычных составах с использованием предобученных FastText‑эмбеддингов.

## Основные возможности

- Получение информации о продукте по штрих-коду через абстрактный провайдер (по умолчанию Open Food Facts API)
- Распознавание в тексте ингредиентов следующих категорий:  
  `lactose` (лактоза), `gluten` (глютен), `egg` (яйца), `meat` (мясо), `fish` (рыба), `shellfish` (моллюски), `peanut` (арахис), `tree_nut` (древесные орехи), `sesame` (кунжут), `soy` (соя), `sodium` (соль/натрий), `sugar` (сахар)
- Возврат позиций каждого найденного аллергена в строке состава
- REST API на Flask:
  - `GET /barcode/<code>` – полная информация о продукте + выделенные аллергены
  - `POST /ingredients` – анализ произвольного текста состава
- Поддержка CPU и GPU (автоматический выбор, рекомендуется CPU для низкой задержки)

## Установка и запуск

### 1. Клонирование

```bash
git clone https://github.com/TeaShpoon/ScanDietBackend.git
cd ScanDietBackend
```

### 2. Зависимости

Рекомендуется использовать виртуальное окружение (Python 3.14+).

```bash
pip install -r requirements.txt
```

### 3. Запуск

```bash
python main.py [--host HOST] [--port PORT] [--debug] [--device DEVICE] [model_path]
```

**Аргументы командной строки:**

| Параметр     | По умолчанию | Описание |
|--------------|--------------|----------|
| `model_path` | `./model.pt` | Путь к файлу модели `.pt` |
| `--host`     | `localhost`  | IP-адрес для прослушивания |
| `--port`     | `3000`       | Порт |
| `--device`   | `auto`       | Устройство инференса: `cpu`, `cuda` или `auto` |
| `--debug`    | выкл.        | Включить отладочный режим Flask |

**Пример запуска на всех интерфейсах:**

```bash
python main.py --host 0.0.0.0 --port 8080 --device cpu ./models/model_bilstm_crf.pt
```

## REST API

### 1. Информация о продукте по штрих-коду

```http
GET /barcode/2100100030001
```

**Ответ (200 OK):**

```json
{
  "name": "Молоко 3,2%",
  "ingredients": "Нормализованное молоко.",
  "labels": {
    "lactose": [
      [
        16,
        22
      ]
    ]
  }
}
```

- `allergens` – словарь, где ключ – код аллергена (совпадает с классом модели), значение – список интервалов `[start, end]` в строке `ingredients` (координаты символов).
- Если продукт не найден, возвращается `404` с описанием ошибки.

### 2. Анализ произвольного текста состава

```bash
curl -X POST localhost:3000/ingredients -d "Яичный желток, масло, соль, лимонный сок"
```

**Ответ (200 OK):**

```json
{
  "ingredients": "Яичный желток, масло, соль, лимонный сок",
  "labels": {
    "egg": [
      [
        7,
        13
      ]
    ],
    "sodium": [
      [
        22,
        26
      ]
    ]
  }
}
```

## Обучение собственной модели

Весь процесс обучения модели вынесен в директорию `train/`.  
Там находятся:

  - `allergens.conll` – размеченный датасет в формате CoNLL (токен, BIO‑метка)
  - `BiLSTM_CRF_Fasttext.ipynb` – Jupyter ноутбук, который:
  - загружает и предобрабатывает данные
  - строит словарь, инициализирует FastText эмбеддинги
  - создаёт и обучает модель BiLSTM‑CRF
  - оценивает качество на тестовой выборке
  - экспортирует обученную модель в TorchScript (`.pt`)

**Как обучить свою модель:**

1. Убедитесь, что установлены Jupyter и все зависимости из `requirements.txt`.
2. Скачайте файл `cc.ru.300.bin.gz` с официального сайта [FastText](https://fasttext.cc/docs/en/crawl-vectors.html) и разархивируйте его.  
3. Полученный файл `cc.ru.300.bin` поместите в корень проекта (он потребуется и для обучения, и для инференса).
4. Запустите ноутбук:
   ```bash
   jupyter notebook train/train_model.ipynb
   ```
5. Выполните ячейки последовательно. При необходимости измените параметры обучения (batch size, learning rate и т.п.) прямо в ноутбуке.
6. После завершения в папке `train/` или корне проекта появится файл модели. Скопируйте его в корень сервера или укажите путь при запуске.

Ноутбук содержит подробные комментарии на русском языке, поясняющие каждый шаг.
